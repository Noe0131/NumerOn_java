# NumerOn_java
ランダムに生成する3桁の数字を当てる • 各桁は0～9。ただし、同じ数字が2回現れることは無い（例："224"や"373"等は生成 されない） 。3桁なので、100の位は0以外。 • プレイヤーは生成された数字を推理し、3桁の数字を入力。 • 数字を比較し、数字と桁が合っていれば「EAT」(イート)、数字はあってるが桁は 合ってない場合は「BITE」(バイト)となる。例えば生成値が"765"で入力値が"746"の 場合、"7"はEAT、"6"はBITEとなるので、「1EAT-1BITE」となる。 • 10回以内に当てられなければゲームオーバー。 • 途中でギブアップする場合は”g”と入力。 • ゲームオーバーになったら、続けるかどうかを問い合わせる。続ける場合は3桁の数 字は新たに生成する。
